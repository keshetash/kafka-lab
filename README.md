# Kafka Lab — Сервис по ремонту техники

Лабораторная работа по потоковой обработке данных в реальном времени с использованием Apache Kafka.

## Описание

Реализован полный цикл работы с сообщениями в Kafka на примере сервиса по ремонту техники. Producer генерирует заказы на ремонт (клиент, устройство, тип неисправности, инженер, запчасть, статус, стоимость) и отправляет их в топик. Consumer читает сообщения из топика, валидирует их и выводит результат в консоль.

Первое сообщение намеренно генерируется с невалидными данными для демонстрации работы валидации.

## Стек технологий

* **Apache Kafka 4.2.0** (KRaft mode, без ZooKeeper) — брокер сообщений
* **Python 3** — язык разработки скриптов
* **kafka-python** — клиентская библиотека для работы с Kafka из Python

## Структура проекта

```
kafka-lab/
├── message\_generator.py  # Генерация случайных сообщений (вне Producer — принцип SOLID)
├── producer.py           # Отправка сообщений в Kafka-топик
├── consumer.py           # Чтение и валидация сообщений из Kafka-топика
└── requirements.txt      # Зависимости Python
```

## Формат сообщения

```json
{
  "order\_id": "RP-4821",
  "client": "Иванов А.С.",
  "device\_type": "Смартфон",
  "fault\_type": "Треснул экран",
  "engineer": "Смирнов И.В.",
  "part": "Дисплей",
  "status": "В работе",
  "price": 3500,
  "timestamp": "2026-03-13T14:22:04.123456"
}
```

## Правила валидации (Consumer)

|Поле|Условие невалидности|
|-|-|
|`order\_id`|не начинается с `RP-` или длина меньше 7 символов|
|`status`|не входит в список допустимых значений|
|`price`|не целое число или выходит за диапазон 1–100000|
|`client`|пустая строка|
|`engineer`|пустая строка|
|обязательные поля|отсутствует любое из 9 полей|

## Запуск

### Требования

* Java 17+
* Python 3.8+
* Apache Kafka 4.2.0

### 1\. Установить зависимости Python

```
pip install -r requirements.txt
```

### 2\. Инициализировать Kafka (только один раз)

```
cd C:\\path\\to\\kafka

bin\\windows\\kafka-storage.bat random-uuid
bin\\windows\\kafka-storage.bat format --standalone -t <UUID> -c config\\server.properties
```

### 3\. Запустить Kafka (окно 1)

```
bin\\windows\\kafka-server-start.bat config\\server.properties
```

### 4\. Запустить Consumer (окно 2)

```
python consumer.py
```

### 5\. Запустить Producer (окно 3)

```
python producer.py
```
